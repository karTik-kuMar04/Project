# Domain docs

This is a single-context repository.

## Before exploring

- Read `CONTEXT.md` at the repository root when it exists.
- Read relevant architectural decisions under `docs/adr/` when that directory exists.
- If either is absent, proceed silently. Domain documentation is created only when decisions need to be recorded.

## Vocabulary

Use the terms defined in `CONTEXT.md` consistently. If a required concept is missing, avoid inventing competing terminology and note the gap for a future documentation session.

## Architectural decisions

Surface any conflict with an existing ADR explicitly rather than silently overriding it.
